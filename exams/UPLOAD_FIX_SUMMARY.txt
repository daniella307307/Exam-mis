🔧 PDF UPLOAD FIX - BUNNY CDN WITH LOCAL FALLBACK
=================================================

❌ PROBLEM:
-----------
- Upload was failing with: "Failed to upload to Bunny CDN. HTTP Status: 0"
- Root cause: Server cannot reach external Bunny CDN (no internet connectivity or network restriction)
- Error in logs: "Could not resolve host: bluelakes.storage.bunnycdn.com"

✅ SOLUTION IMPLEMENTED:
--------------------------
The upload handler now has a **two-tier strategy**:

1. PRIMARY: Try Bunny CDN upload first
   - Attempts to upload to: bluelakes.storage.bunnycdn.com
   - If successful → Returns Bunny CDN URL
   - If failed → Falls back to local storage

2. FALLBACK: Local file storage
   - Stores PDF locally in: /var/www/html/Exam-mis/exams/practical_exams/
   - Organized by school: practical_exams/[school_name]/
   - Returns local URL: /Exam-mis/exams/practical_exams/[school]/file.pdf
   - Accessible via web browser

🎯 WORKFLOW - HOW IT WORKS NOW:
--------------------------------

STEP 1: Teacher selects "Upload PDF Practical"
STEP 2: Chooses PDF file and clicks upload
STEP 3: System attempts Bunny upload with timeout (10 sec connect, 30 sec total)
STEP 4A: If Bunny succeeds → URL like: https://bluelakes1988.b-cdn.net/exams/School/file.pdf
STEP 4B: If Bunny fails → URL like: http://localhost/Exam-mis/exams/practical_exams/School/file.pdf
STEP 5: Exam published with whichever URL was successful
STEP 6: Students access via the returned URL (doesn't matter if Bunny or local)

🔍 RESPONSE FORMAT:
--------------------
Success response includes "storage" field:

{
  "success": true,
  "url": "https://bluelakes1988.b-cdn.net/exams/School_Name/2026-04-18_154530_exam.pdf",

  "url": "/Exam-mis/exams/practical_exams/School_Name/2026-04-18_154530_exam.pdf",

             OR "PDF uploaded successfully (stored locally - ensure Bunny CDN connectivity for production)"
}
